## 1. 알림장 개요

### 1.1 개념 정의

- **알림장(Notice Board)**은 교사 ↔ 학부모 간 일상 커뮤니케이션 채널이다.
- **보낸 알림장**: 교사가 작성하여 학부모에게 전달하는 알림장 (기존 구조 유지)
- **받은 알림장**: 학부모가 교사에게 작성하여 보낸 상황 전달 메시지 (신규 구조)
- 두 방향의 알림장은 **받은 알림장 / 보낸 알림장 탭**으로 화면에서 구분한다.

### 1.2 비즈니스 규칙 요약

```
보낸 알림장 (notice_boards) ──1:N──→ notice_children ──1:N──→ notice_comments
받은 알림장 (parent_notices) ──N:1──→ children · 학부모 단방향 메시지
```

| 규칙 | 내용 |
|------|------|
| 보낸 알림장 | 교사 1회 작성 → 반 전체 또는 선택 원아에게 동시 발행 |
| 받은 알림장 | 학부모가 특정 카테고리(결석/건강/일정/준비물)로 교사에게 전달 |
| 댓글 격리 | `notice_child_id` 기준 — 원아 간 댓글 노출 불가 |
| 수정 범위 | 교사: 본인 작성 알림장 수정 / 기관관리자: 전체 수정 |
| 삭제 | 교사: 본인만 / 기관관리자 이상: 모두 |
| 발행 상태 | `published` → `deleted` (삭제 = 회수) / `scheduled` 예약 (`draft`는 리스트 미노출 — DRAFT_STORAGE 별도 관리) |
| AI 기능 | 예문 자동생성, 문장 다듬기(교정) |

> ⚠️ **원아별 보기(child view)는 제거됨** — 받은 알림장 / 보낸 알림장 탭 구조로 대체

## 2. 작성 모드 (핵심 변경 — 2026-03-27)

### 2.1 개요

알림장 작성은 **단일작성**과 **다중작성** 두 가지 모드로 분리된다.

| 항목 | 단일작성 (Single) | 다중작성 (Multi) |
|------|-------------------|-----------------|
| 관계 | 교사 1 : 학부모 1 | 교사 1 : 학부모 N |
| 원아 선택 | 1명만 선택 가능 | 복수 선택 가능 |
| 원아별 입력 | **종합평가** (6개 카테고리) + 개별메시지 | **태그 선택** (카테고리별 칩) + 개별메모 |
| 사진/동영상 | 포스트 레벨 (공유) | 포스트 레벨 (공유) |
| 목록 카드 표시 | 원아 1명 강조 표시 + `1:1` 뱃지 | 원아 수 칩 + `다중` 뱃지 |
| `writeType` 필드 | `'single'` | `'multi'` |

### 2.2 모드 전환 UI

- 작성 패널 상단에 **모드 탭 바** (`.write-mode-bar`)
- `단일작성` / `다중작성` 탭 클릭으로 전환
- 전환 시 원아 선택·상태 초기화
- **수정 모드(`isEditMode`)에서는 모드 전환 불가** — 버튼 비활성(disabled) + 토스트 안내

### 2.3 단일작성 모드 (Single)

#### 원아 선택
- 원아 목록에서 **1명만** 선택 가능 (다른 원아 클릭 시 기존 선택 해제)

#### 종합평가 (daily_eval)
원아 카드 내 6개 카테고리를 3단계(좋음/보통/나쁨)로 평가:

| 카테고리 | 필드 | 옵션 | 확장 표시 조건 | 확장 내용 |
|---------|------|------|:---:|---------|
| 기분 | `evalMood` | 좋음/보통/나쁨 | — | — |
| 건강 | `evalHealth` | 좋음/보통/나쁨 | **나쁨** 선택 시 | 체온 입력 + 증상 칩(복수) + 추가 메모 |
| 식사량 | `evalMeal` | 좋음/보통/나쁨 | **나쁨** 선택 시 | 추가 메모 textarea |
| 배변 | `evalToilet` | 좋음/보통/나쁨 | **항상** | 배변 상태(단일) + 횟수 입력 + 메모 |
| 참여도 | `evalActivity` | 좋음/보통/나쁨 | **선택 시** | 메모 textarea |
| 교우관계 | `evalSocial` | 좋음/보통/나쁨 | **선택 시** | 관찰 메모 textarea |

**건강 증상 칩**: 발열 / 기침 / 콧물 / 설사 / 구토 / 복통 / 두통 / 알레르기

**배변 상태 옵션**: 정상 / 딱딱함 / 묽음 / 설사 / 배변없음

#### 요약 생성 규칙
```
좋음 ≥ 4개 & 나쁨 = 0개 → "아주 좋아요 👍"
나쁨 ≥ 1개              → "확인이 필요해요 ⚠️"
그 외                   → "무난해요 😊"
```

### 2.4 다중작성 모드 (Multi)

#### 원아 선택
- 복수 선택 가능 (체크박스 방식)

#### 태그 시스템
6개 카테고리 × 칩 선택 방식:

| 카테고리 | 태그 칩 |
|---------|---------|
| 감정 | 활발했어요, 기뻐했어요, 자신감있었어요, 수줍어했어요, 졸려했어요, 울었어요, 차분했어요 |
| 활동 | 블록놀이, 그림그리기, 색칠하기, 역할놀이, 율동, 독서, 퍼즐, 점토놀이 |
| 건강 | 양호, 기침, 콧물, 발열, 배탈, 알레르기 |
| 식사 | 잘먹음, 조금먹음, 안먹음, 편식함, 간식좋아함 |
| 수면 | 잘잠, 조금잠, 못잠, 안잠 |
| 기타 | 친구와놀이, 선생님과대화, 혼자놀이, 야외산책, 특별활동 |

- **카테고리 탭** (`.tag-cat-tabs`)으로 카테고리 전환 — **원아별 독립 상태** (`writeTagActiveCats[cid]`)
- **칩 그리드** (`.tag-chip-grid`)에서 복수 선택
- 선택된 태그는 칩 위에 표시 + `×` 버튼으로 개별 제거
- **선택값 있는 카테고리**: 탭 배경색 연하게 변경 (`.has-tags`) + 작은 도트(●) 표시
- **전체 적용 버튼**: 한 원아의 태그를 나머지 원아 전체에 복사
- **태그 초기화**: 원아 카드 우측상단 텍스트링크 (태그 1개 이상일 때만 표시)

#### 개별메모
- textarea로 원아별 개별 메모 입력

## 3. 사진/동영상 (포스트 레벨)

### 3.1 구조 변경
- **기존**: 원아별 사진 첨부
- **변경**: **포스트(알림장) 레벨** 사진/동영상 첨부 (모든 원아 공유)

### 3.2 미디어 스테이지
- 작성 패널에 **미디어 스테이지 영역** (`.media-stage`)
- `📷 사진 추가` / `🎬 동영상 추가` 버튼 → 클릭 시 모의 이미지/동영상 스테이지에 추가
- 스테이지에 썸네일 그리드 표시
- 각 썸네일: 삭제 버튼(`×`), 타입 뱃지(📷/🎬), **순서변경 ◀▶ 버튼** (호버 시 표시)

### 3.3 대표 이미지
- 스테이지에서 이미지 **클릭** → 대표 이미지로 지정
- 대표 이미지: 금색 테두리 + ⭐ 뱃지
- 알림장 목록 카드에 대표 이미지 썸네일 표시

### 3.4 데이터 구조
```javascript
writePhotos = [
  { url: '...', type: 'photo', isRepresentative: true },
  { url: '...', type: 'video', isRepresentative: false }
];
```

### 8.4 작성 패널 닫기 확인

패널을 닫는 모든 경로(X버튼, 취소, ESC, 오버레이 클릭)에서 **dirty 체크** 후 분기:

| 상태 | 동작 |
|------|------|
| dirty 없음 | 확인 없이 즉시 닫기 |
| dirty 있음 | 확인 모달 표시 |

**dirty 조건**: 반 선택됨 OR 원아 선택됨 OR 본문 입력됨

**확인 모달 버튼:**
- `계속 작성` — 모달 닫고 패널 유지
- `그냥 닫기` — 내용 버리고 즉시 닫기 (빨간 버튼)
- `🗂️ 임시저장 후 닫기` — saveDraft() 호출 후 닫기 (primary 버튼)

> 발행·예약발행·`saveDraft()` 내부에서 호출하는 `closeWritePanel()`은 dirty 체크 없이 강제 닫기.
