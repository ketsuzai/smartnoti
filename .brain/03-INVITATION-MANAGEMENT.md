## 1. 초대장 관리 개요

초대장 관리는 **2개의 대탭**과 **학부모/교직원 서브탭**으로 구성:

```
📋 초대장 관리
├── [탭1] 초대 발송 관리
│   ├── [서브탭] 학부모 초대장
│   │   ├── 기관 선택 → 반별 초대장 관리
│   │   └── 초대 링크 생성/중지/재개
│   └── [서브탭] 교직원 초대장
│       └── 기관 선택 → 반-역할별 초대 관리
└── [탭2] 초대 이력 조회
    └── 전체 초대 이력 테이블
```

## 2. 초대 발송 관리 — 학부모 초대장

### 2.1 기관 선택 화면
- 기관 카드 그리드 형태로 표시
- 기관별 초대 현황 (전체 반 수, 활성 반 수, 총 회원 수, 대기 회원 수)
- 검색: 기관명/원장명

#### 기관 카드 데이터
```javascript
{
  id: Number,
  name: String,
  type: String,         // '어린이집'|'유치원'
  region: String,
  director: String,
  totalClasses: Number,
  activeClasses: Number,
  totalMembers: Number,
  pendingMembers: Number
}
```

### 2.2 반별 초대장 관리 (기관 진입 후)

#### 반(Class) 데이터
```javascript
{
  id: Number,
  orgId: Number,
  name: String,          // 반 이름 (햇님반, 새싹반 등)
  teacher: String,       // 담당 교사
  memberCount: Number,   // 현재 가입 학부모 수
  capacity: Number,      // 정원
  status: String,        // 'active'|'stopped'|'none'
  link: String,          // 초대 링크 URL
  toggleOn: Boolean,     // 초대 활성화 토글 상태
  members: Array<{       // 가입한 학부모 목록
    name: String,
    child: String,
    date: String,
    av: String,
    vc: String
  }>
}
```

#### 초대장 상태
| 상태 | 설명 | UI 표시 |
|------|------|---------|
| `active` | 초대 링크 활성 | 초록 뱃지, 토글 ON |
| `stopped` | 초대 일시 중지 | 노랑 뱃지, 토글 OFF |
| `none` | 초대 미설정 | 회색 뱃지, 미생성 상태 |

#### 기능
- **초대 링크 생성**: 미설정 반에 대해 새 링크 생성
- **초대 링크 복사**: 클립보드에 링크 복사
- **초대 중지/재개**: 토글로 ON/OFF 전환
- **초대 삭제**: 확인 모달 후 삭제
- **가입 학부모 목록 보기**: 반별 가입 완료 학부모 확인

## 3. 초대 발송 관리 — 교직원 초대장

### 3.1 교직원 초대 데이터
```javascript
{
  id: String,
  orgId: Number,
  cls: String,           // 반 이름
  role: String,          // '담임교사'|'보조교사'
  invitedBy: String,     // 초대 발송자
  sentAt: String,        // 발송일 (YYYY-MM-DD)
  status: String,        // 'pending'|'used'|'expired'
  dDay: Number,          // D-day (양수: 남은 일, 음수: 만료 지남)
  name: String,          // 가입자 이름 (가입 완료 시)
  link: String           // 초대 링크
}
```

### 3.2 교직원 초대 상태
| 상태 | 설명 |
|------|------|
| `pending` | 발송 완료, 가입 대기 |
| `used` | 초대 수락 → 가입 완료 |
| `expired` | 기한 만료 |

## 4. 초대 이력 조회 (탭2)

### 4.1 이력 데이터 모델
```javascript
{
  id: Number,
  status: String,        // 'active'|'used'|'expired'|'stopped'
  type: String,          // 'parent'|'teacher'
  org: String,           // 기관명
  cls: String,           // 반 이름
  name: String,          // 학부모/교사 이름
  channel: String,       // 발송 채널: '카카오'|'문자'|'링크복사'
  sentAt: String,        // 발송 일시
  joinedAt: String,      // 가입 일시 (미가입 시 빈 문자열)
  link: String,          // 초대 링크
  _sel: Boolean          // UI 선택 상태
}
```

### 4.2 KPI 카드
| 카드 | 색상 | 설명 |
|------|------|------|
| 전체 초대 | 회색 | 총 초대 발송 건수 |
| 활성(가입완료) | 초록 | 초대 수락 후 가입 완료 |
| 미가입(대기/사용완료) | 파랑 | 사용 후 아직 미가입 |
| 만료 | 빨강 | 기한 만료된 초대 |

### 4.3 이력 필터
- **상태**: 전체, 활성, 사용완료, 만료, 중지
- **유형**: 전체, 학부모, 교직원
- **기관 필터**: 기관명 검색
- **텍스트 검색**: 이름, 기관명

### 4.4 이력 테이블 컬럼
`체크박스 | 상태 | 유형 | 기관명 | 반명 | 이름 | 발송채널 | 발송일시 | 가입일시 | 초대링크 | 관리`

### 4.5 이력 관리 액션
| 액션 | 설명 |
|------|------|
| 재발송 | 초대 재발송 |
| 만료처리 | 활성 초대를 수동 만료 |
| 삭제 | 이력 삭제 |

## 5. 운영관리 초대 관리 (`operation-invitation.html`) — v1 스펙 (2026-03-30 확정)

### 5.1 개요

기관 컨텍스트 안에서 동작. 기관 선택 단계 없음.
탭 구조: `[📩 학부모 초대] [👨‍🏫 교직원 초대] [📋 초대 이력]`

### 5.2 역할별 접근 범위

| 역할 | 학부모 초대 | 교직원 초대 | 초대 이력 |
|------|:---:|:---:|:---:|
| super (운영관리 컨텍스트) | ✅ 전체 반 카드 | ✅ | ✅ 전체 |
| org_admin | ✅ 전체 반 카드 | ✅ | ✅ 전체 |
| teacher | ✅ 담당 반 카드만 | ❌ (탭 disabled + toast) | ✅ 담당 반만 |

---

### 5.3 탭1 — 학부모 초대 (반 단위 카드 그리드)

#### 5.3.1 카드 그리드 구조

2열 그리드. 1카드 = 1반.

```
┌─────────────────────┐  ┌─────────────────────┐
│ 햇님반        [활성] │  │ 달님반      [중지됨] │
│ 만5세 · 담임: 이지현  │  │ 만4세 · 담임: 박수진  │
│ 가입 학부모  18/20명 │  │ 가입 학부모  12/18명 │
│ ████████░░ 90%      │  │ ███████░░░ 67%      │
│ [링크URL]  [복사]   │  │ [링크URL]  [복사]   │
│ ●ON  👥18명  [중지][삭제]│ │ ●OFF 👥12명 [재개][삭제]│
└─────────────────────┘  └─────────────────────┘
┌─────────────────────┐
│ 별빛반      [미설정] │
│ 만3세 · 담임: 최민준  │
│ 가입 학부모    0/15명│
│ ░░░░░░░░░░ 0%       │
│ [초대 링크 미생성]   │
│ 링크 없음  [링크 생성]│
└─────────────────────┘
```

#### 5.3.2 반 상태값

| 상태 | 설명 | 뱃지 | 토글 | 푸터 액션 |
|------|------|------|------|----------|
| `active` | 초대 링크 활성 | 초록 "활성" | ON | [중지] [삭제] |
| `stopped` | 초대 일시 중지 | 주황 "중지됨" | OFF | [재개] [삭제] |
| `none` | 초대 미설정 | 회색 "미설정" | 없음 | [링크 생성] |

#### 5.3.3 액션

| 액션 | 설명 |
|------|------|
| 링크 복사 | 클립보드에 링크 복사 + 토스트 |
| 토글 ON→OFF | status: `active`→`stopped` |
| 토글 OFF→ON | status: `stopped`→`active` |
| [중지] / [재개] | 토글과 동일 동작 |
| [삭제] | status→`none`, link→null |
| [링크 생성] | status→`active`, 링크 자동 생성 |
| 👥 N명 가입 칩 | 가입 학부모 목록 모달 오픈 |

#### 5.3.4 새 초대 링크 생성 모달

```
┌─────────────────────────────┐
│  🔗 초대 링크 생성           │
├─────────────────────────────┤
│  대상 반   [select: none반만]│
│  만료 기간 [select]          │  → 만료없음/7일/14일/30일
├─────────────────────────────┤
│  [취소] [생성 및 링크복사] [생성]│
└─────────────────────────────┘
```

- `none` 상태 반만 목록에 표시
- [생성 및 링크복사]: 생성 후 클립보드 자동 복사

### 5.4 탭2 — 교직원 초대

#### 5.4.1 교직원 초대 테이블 컬럼

`상태 | 반 | 역할 | 발송자 | D-Day | 가입자 | 관리`

#### 5.4.2 상태값

| 상태 | 설명 | 뱃지 색 | 관리 버튼 |

#### 5.4.3 교직원 초대 생성 모달

## 6. 초대 관련 비즈니스 규칙

| 규칙 | 내용 |
|------|------|
| 학부모 초대 단위 | **반(班) 단위** (1반 = 1링크) |
| 링크 만료 | 없음(상시) 또는 7/14/30일 선택 |
| 좌석 수 | 학부모: 제한 없음 / 교직원: 기본 1, 최대 10 |
| 교직원 초대 | 반별 단일 초대장, D-day 관리 |
| 교사 권한 | 담당 반 카드만 보임, 교직원 탭 접근 불가 |
