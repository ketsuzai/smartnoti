# 🧠 프로젝트 브레인: 초대장 관리 기능정의

> **최종 업데이트**: 2026-03-23
> **관련 파일**: `invitation-admin.html` (통합관리자 전용), `src/pages/oper/operation-invitation.html` (운영관리 — 기관관리자/교사/통합관리자 운영컨텍스트)

---

## 1. 초대장 관리 개요

초대장 관리는 **2개의 대탭**과 **학부모/교직원 서브탭**으로 구성:

```
📋 초대장 관리
├── [탭1] 초대 발송 관리
│   ├── [서브탭] 학부모 초대장
│   │   ├── 기관 선택 → 반별 초대장 관리
│   │   └── 초대 링크 생성/중지/재개
│   └── [서브탭] 교직원 초대장
│       └── 기관 선택 → 반-역할별 초대 관리
└── [탭2] 초대 이력 조회
    └── 전체 초대 이력 테이블
```

---

## 2. 초대 발송 관리 — 학부모 초대장

### 2.1 기관 선택 화면
- 기관 카드 그리드 형태로 표시
- 기관별 초대 현황 (전체 반 수, 활성 반 수, 총 회원 수, 대기 회원 수)
- 검색: 기관명/원장명

#### 기관 카드 데이터
```javascript
{
  id: Number,
  name: String,
  type: String,         // '어린이집'|'유치원'
  region: String,
  director: String,
  totalClasses: Number,
  activeClasses: Number,
  totalMembers: Number,
  pendingMembers: Number
}
```

### 2.2 반별 초대장 관리 (기관 진입 후)

#### 반(Class) 데이터
```javascript
{
  id: Number,
  orgId: Number,
  name: String,          // 반 이름 (햇님반, 새싹반 등)
  teacher: String,       // 담당 교사
  memberCount: Number,   // 현재 가입 학부모 수
  capacity: Number,      // 정원
  status: String,        // 'active'|'stopped'|'none'
  link: String,          // 초대 링크 URL
  toggleOn: Boolean,     // 초대 활성화 토글 상태
  members: Array<{       // 가입한 학부모 목록
    name: String,
    child: String,
    date: String,
    av: String,
    vc: String
  }>
}
```

#### 초대장 상태
| 상태 | 설명 | UI 표시 |
|------|------|---------|
| `active` | 초대 링크 활성 | 초록 뱃지, 토글 ON |
| `stopped` | 초대 일시 중지 | 노랑 뱃지, 토글 OFF |
| `none` | 초대 미설정 | 회색 뱃지, 미생성 상태 |

#### 기능
- **초대 링크 생성**: 미설정 반에 대해 새 링크 생성
- **초대 링크 복사**: 클립보드에 링크 복사
- **초대 중지/재개**: 토글로 ON/OFF 전환
- **초대 삭제**: 확인 모달 후 삭제
- **가입 학부모 목록 보기**: 반별 가입 완료 학부모 확인

## 3. 초대 발송 관리 — 교직원 초대장

### 3.1 교직원 초대 데이터
```javascript
{
  id: String,
  orgId: Number,
  cls: String,           // 반 이름
  role: String,          // '담임교사'|'보조교사'
  invitedBy: String,     // 초대 발송자
  sentAt: String,        // 발송일 (YYYY-MM-DD)
  status: String,        // 'pending'|'used'|'expired'
  dDay: Number,          // D-day (양수: 남은 일, 음수: 만료 지남)
  name: String,          // 가입자 이름 (가입 완료 시)
  link: String           // 초대 링크
}
```

### 3.2 교직원 초대 상태
| 상태 | 설명 |
|------|------|
| `pending` | 발송 완료, 가입 대기 |
| `used` | 초대 수락 → 가입 완료 |
| `expired` | 기한 만료 |

## 4. 초대 이력 조회 (탭2)

### 4.1 이력 데이터 모델
```javascript
{
  id: Number,
  status: String,        // 'active'|'used'|'expired'|'stopped'
  type: String,          // 'parent'|'teacher'
  org: String,           // 기관명
  cls: String,           // 반 이름
  name: String,          // 학부모/교사 이름
  channel: String,       // 발송 채널: '카카오'|'문자'|'링크복사'
  sentAt: String,        // 발송 일시
  joinedAt: String,      // 가입 일시 (미가입 시 빈 문자열)
  link: String,          // 초대 링크
  _sel: Boolean          // UI 선택 상태
}
```

### 4.2 KPI 카드
| 카드 | 색상 | 설명 |
|------|------|------|
| 전체 초대 | 회색 | 총 초대 발송 건수 |
| 활성(가입완료) | 초록 | 초대 수락 후 가입 완료 |
| 미가입(대기/사용완료) | 파랑 | 사용 후 아직 미가입 |
| 만료 | 빨강 | 기한 만료된 초대 |

### 4.3 이력 필터
- **상태**: 전체, 활성, 사용완료, 만료, 중지
- **유형**: 전체, 학부모, 교직원
- **기관 필터**: 기관명 검색
- **텍스트 검색**: 이름, 기관명

### 4.4 이력 테이블 컬럼
`체크박스 | 상태 | 유형 | 기관명 | 반명 | 이름 | 발송채널 | 발송일시 | 가입일시 | 초대링크 | 관리`

### 4.5 이력 관리 액션
| 액션 | 설명 |
|------|------|
| 재발송 | 초대 재발송 |
| 만료처리 | 활성 초대를 수동 만료 |
| 삭제 | 이력 삭제 |

## 5. 운영관리 초대 관리 (`operation-invitation.html`) — 2026-03-23 신규 구현

### 5.1 개요

`invitation-admin.html`(통합관리자 전용)과 별개로, **운영관리 섹션 전용 초대 관리 페이지**.
기관 컨텍스트 안에서 동작하므로 기관 선택 단계 없음.

### 5.2 역할별 접근 범위

| 역할 | 학부모 초대 | 교직원 초대 | 초대 이력 |
|------|:---:|:---:|:---:|
| super (운영관리 컨텍스트) | ✅ 전체 반 | ✅ | ✅ |
| org_admin | ✅ 전체 반 | ✅ | ✅ |
| teacher | ✅ **담당 반만** | ❌ 탭 비활성 | ✅ 담당 반 이력만 |

- teacher 역할: 교직원 초대 탭 클릭 시 "권한이 없습니다" toast 표시

### 5.3 탭 구조

```
[📩 학부모 초대] [👨‍🏫 교직원 초대] [📋 초대 이력]
```

### 5.4 화면 구성

**공통 상단 KPI 4개**
| 카드 | 색상 | 내용 |
|------|------|------|
| 전체 초대 발송 | --accent | MOCK_HISTORY 전체 건수 |
| 가입 완료 | --green | status: active 건수 |
| 대기 중 | --amber | status: pending 건수 |
| 만료됨 | --red | status: expired 건수 |

**탭1 — 학부모 초대장**
- 반별 카드 리스트 (진행바, 링크, 토글 ON/OFF, 삭제)
- 카드 클릭 → 우측 Drawer (링크 박스 + 가입 학부모 목록)
- [+ 새 초대 링크 생성] 모달 (반 선택 → 링크 생성)

**탭2 — 교직원 초대장** (org_admin/super만)
- 테이블: 상태 · 반 · 역할 · 발송자 · D-day · 가입자 · 링크 · 관리
- [+ 교직원 초대 발송] 모달 (반 선택 + 역할 버튼 그룹)

**탭3 — 초대 이력**
- 필터: 상태 / 유형 / 반 / 이름검색 / 초기화
- 테이블: 체크박스 · 상태 · 유형 · 반 · 이름 · 발송채널 · 발송일시 · 가입일시 · 링크 · 관리
- 체크박스 일괄 삭제 지원

### 5.5 모달

| 모달 | 설명 |
|------|------|
| 새 초대 링크 생성 | 반 선택 → mock 링크 생성, status → 'active' |
| 교직원 초대 발송 | 반 선택 + 역할(담임/보조) → 목록 추가 |
| 삭제 확인 | danger 스타일 confirm |

## 6. 초대 관련 비즈니스 규칙

| 규칙 | 설명 |
|------|------|
| 초대 유효기간 | D-Day 카운트다운 방식 (일 단위) |
| 자동 만료 | D-Day 0 이하 시 expired 처리 |
| 채널 유형 | 카카오톡, 문자(SMS), 링크복사 3가지 |
| 학부모 초대 단위 | 반(Class) 단위로 초대 링크 생성 |
| 교직원 초대 단위 | 반+역할 조합 단위로 초대 |
| 중복 초대 방지 | 동일 반에 중복 링크 생성 불가 (추정) |
