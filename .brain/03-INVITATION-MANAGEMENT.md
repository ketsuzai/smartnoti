# 프로젝트 브레인: 초대 관리 기능 정의

> 최종 업데이트: 2026-04-17
> 기준 화면: `src/pages/oper/operation-invitation.html`
> 공통 정책 반영 대상: `src/pages/invitation-admin.html`, `src/pages/oper/operation-invitation.html`

---

## 1. 개요

초대 관리는 학부모 초대와 교직원 초대로 구분한다.

- 학부모 초대: 반별 마스터 링크 기반 운영
- 교직원 초대: 개별 링크 생성 기반 운영
- 초대이력: 기존 구조 유지, 명칭만 `완료`에서 `초대이력`으로 통일

이번 개정의 핵심은 아래 3가지다.

1. 학부모 초대는 `반별 공용 링크 + 가입 완료 멤버 리스트` 구조로 고정한다.
2. 교직원 초대는 `생성 단계 -> 생성 완료/링크 확인 단계 -> 리스트 관리` 구조로 정리한다.
3. `operation-invitation`에서 먼저 구현하되, 정책 문구와 상태 규칙은 `invitation-admin`에도 공통 반영한다.

## 2. 공통 정책

### 2.1 공통 상태 언어

- 활성 상태 문구: `초대 가능`
- 비활성 상태 문구: `초대 중지`

### 2.2 공통 링크 운영 정책

- 링크는 복사 가능해야 한다.
- 링크 만료 처리는 확인 팝업 후 실행한다.
- 링크 만료 처리 완료 시 성공 토스트를 노출하고 상태 배지를 즉시 `초대 중지`로 변경한다.
- 링크 재생성은 확인 팝업 후 실행한다.
- 링크 재생성 완료 시 새 링크로 즉시 교체한다.
- 링크 재생성 완료 후 성공 토스트를 노출한다.
- 링크 재생성 완료 후 `기존 링크는 만료되었으며 새 링크를 다시 공유해야 합니다` 안내를 노출한다.

### 2.3 공통 발송 정책

웹에서 제공하는 공유/발송 방식은 아래 3가지다.

- `카카오톡`
- `문자발송`
- `복사`

발송 흐름은 공통으로 아래 규칙을 따른다.

1. `링크발송` 버튼 클릭
2. 단계형 모달 오픈
3. 채널 선택: `카카오톡 -> 문자발송 -> 복사`
4. `카카오톡` 또는 `문자발송` 선택 시 전화번호 입력
5. 전화번호 입력 필드는 단일 입력칸 1개를 사용하며 자동 하이픈 처리
6. 전화번호 입력 기본값은 비어 있음
7. 확인 단계에서 `전화번호`, `반 이름`, `발송 채널` 표시
8. 확인 시 즉시 발송 처리
9. 성공 토스트 노출

`복사` 선택 시에는 전화번호 입력 없이 즉시 복사와 토스트를 수행한다.

## 3. 학부모 초대

### 3.1 핵심 구조

학부모 초대는 `반별 마스터 링크 1개`를 기준으로 관리한다.

- 반 카드 또는 반 리스트 클릭
- 우측 드로어 오픈
- 드로어 안에서 링크 관리와 가입 완료 멤버 확인을 처리

학부모 드로어의 주요 패턴은 아래 순서로 구성한다.

1. 초대링크 정보
2. 초대링크 액션
3. 초대링크 활성화 토글
4. 가입 멤버 리스트

### 3.2 링크 정보 영역

상단 링크 박스에는 현재 반의 마스터 링크를 노출한다.

기본 안내 문구는 항상 노출한다.

```text
반별 마스터 링크가 생성되었습니다.
링크를 공유하여 학부모님을 일괄 초대할 수 있습니다.
```

상태가 `초대 중지`일 때는 위 기본 안내를 유지하면서 경고 문구를 추가한다.

### 3.3 링크 액션

링크 정보 박스 안 액션 순서는 아래와 같이 고정한다.

1. `복사`
2. `링크발송`
3. `재생성`

세부 규칙은 아래와 같다.

- `복사`: 클립보드에 즉시 복사, `링크가 복사되었습니다` 토스트 노출
- `링크발송`: 공통 발송 정책 사용
- `재생성`: 확인 팝업 후 실행, 새 링크 즉시 교체, 기존 링크 즉시 만료

### 3.4 초대 링크 활성화 토글

- 토글 라벨: `초대 링크 활성화`
- 상태 배지와 토글을 함께 표시
- 토글 On/Off는 즉시 반영
- 별도 확인 팝업 없음
- Off 시 접속자는 `초대 마감` 안내 페이지로 랜딩

토글이 Off여도 아래 액션은 계속 허용한다.

- `복사`
- `링크발송`

단, 상태 배지와 경고 문구를 통해 현재 가입 차단 상태임을 명확히 보여준다.

### 3.5 가입 멤버 리스트

학부모 드로어에서는 `가입 완료 멤버만` 노출한다.

리스트 상단 정보:

- 총 가입 멤버 수
- 정렬 기준: 최신순
- 필요 시 자녀 수는 보조 텍스트로 표시

리스트 항목 규칙:

- 학부모 이름을 주요 정보로 강조
- 자녀 이름은 보조 정보로 노출
- 가입 일시는 우측 정렬
- 각 행 우측에 `내보내기` 버튼 배치

빈 상태 문구는 아래 톤으로 사용한다.

```text
아직 가입한 멤버가 없습니다.
초대 링크를 복사하거나 발송해 학부모를 초대해보세요.
```

### 3.6 내보내기

`내보내기`는 가입 완료 멤버에 대한 사후 보안 관리 액션이다.

- 각 행 우측 개별 버튼으로 제공
- 클릭 시 최종 확인 팝업 노출
- 확인 문구는 운영 안내형 + 되돌릴 수 없음 강조 톤 사용
- 확인 시 해당 멤버를 반 연결에서 강제 제외
- 회원 자체 삭제는 하지 않음
- 초대/연결 데이터는 삭제 처리

## 4. 교직원 초대

### 4.1 핵심 구조

교직원 초대는 개별 링크 생성 기반으로 운영한다.

흐름은 아래 3단계로 고정한다.

1. 생성 단계
2. 생성 완료/링크 확인 단계
3. 초대 리스트 관리

교직원 리스트는 클릭 상세 드로어를 열지 않는다.

### 4.2 생성 단계

생성 단계 입력 항목:

- 소속 반
- 역할
- 링크 생성

역할은 기존 정책을 유지하되 개별 링크 생성 흐름 안에서 처리한다.

### 4.3 생성 완료/링크 확인 단계

핵심 정보는 아래 4개만 우선 노출한다.

- 소속반
- 역할
- 생성된 링크
- 상태 배지

액션 순서는 아래와 같이 고정한다.

1. `복사`
2. `링크발송`
3. `재생성`
4. `링크 만료 처리`

세부 규칙:

- `복사`: 즉시 복사 + 토스트
- `링크발송`: 공통 발송 정책 사용
- `재생성`: 확인 팝업 후 새 링크 발급, 기존 링크 즉시 만료
- `링크 만료 처리`: 확인 팝업 후 상태를 `초대 중지`로 변경

### 4.4 교직원 리스트

교직원 리스트는 운영 액션 중심으로 유지한다.

리스트에서 가능한 액션:

- `복사`
- `링크발송`
- `만료`

액션 순서는 아래와 같이 고정한다.

1. `복사`
2. `링크발송`
3. `만료`

교직원 리스트는 상세 드로어 없이 빠른 운영에 집중한다.
링크 재생성은 생성 완료/링크 확인 단계에서 우선 다룬다.

## 5. 초대이력

### 5.1 기본 원칙

- 탭 명칭은 `완료`가 아니라 `초대이력`으로 통일
- 초대이력 탭 구조는 이번 개편에서 `현재 구조 유지`
- 이번 개편의 주된 변경 범위는 학부모 드로어와 교직원 생성/리스트 영역

### 5.2 역할

초대이력은 결과 확인용 영역으로 사용한다.

- 운영 중인 링크/대상은 리스트 또는 드로어에서 관리
- 초대이력은 기존 정보 구조를 유지하되 명칭과 정책 언어만 공통화

---

## 6. 역할별 범위

### 6.1 운영관리 / 기관관리자

- 학부모 반별 초대 링크 관리 가능
- 학부모 가입 완료 멤버 확인 및 내보내기 가능
- 교직원 개별 링크 생성 가능
- 교직원 링크 발송, 복사, 재생성, 만료 처리 가능

### 6.2 교사

교사의 초대 권한은 기존 정책을 유지한다.

- 학부모 초대는 담당 반 기준으로만 노출
- 교직원 초대는 정책상 제한 가능
- 세부 권한 규칙은 `07-ROLE-PERMISSION.md`와 일치시켜 관리

## 7. 구현 우선순위

1. `operation-invitation.html` 학부모 드로어 구조 개편
2. 학부모 링크 액션: 복사 / 링크발송 / 재생성 / 토글 정리
3. 가입 멤버 리스트 + 내보내기 확인 팝업 정리
4. 교직원 생성 단계와 생성 완료 화면 분리
5. 교직원 리스트 액션 정리
6. `invitation-admin.html`의 공통 정책 문구와 상태 규칙 동기화

## 8. 이번 인터뷰에서 확정된 정책 요약

- 학부모 초대는 반별 마스터 링크 기반
- 학부모 드로어는 가입 완료 멤버만 노출
- 학부모 주요 패턴은 `초대링크(복사) / 초대링크 활성화(토글) / 가입멤버리스트(내보내기)`
- 웹 발송 채널은 `카카오톡 / 문자발송 / 복사`
- `카카오톡 / 문자발송`은 전화번호 입력 모달 후 확인
- 링크 재생성은 확인 팝업 후 실행, 기존 링크 즉시 만료
- 링크 만료 처리는 확인 팝업 후 실행
- 초대이력 탭은 구조 유지, 명칭만 변경
- 정책은 `operation-invitation` 기준으로 확정하되 `invitation-admin`에도 공통 반영
