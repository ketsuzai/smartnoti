## 1. 개요

반관리는 반 단위로 교사·원아를 배치하고, 반 상태(운영/종료/변경)를 관리하는 화면이다.
앱 프론트 기획서(W17)의 기능·정책·플로우를 웹 백오피스 UI로 구현한다.

## 3. 핵심 정책 (앱 기획서 W17 명시)

### 3.1 "삭제"의 의미
- 교사/원아 삭제 = **"해당 반 소속에서 제외"** (계정/데이터 삭제 아님)
- 데이터는 남고, 반에 대한 권한만 해제되는 구조

### 3.2 기록 유지
- 반이동/삭제 후에도 과거 기록(알림장·앨범·댓글 등) 유지
- 원아 상세/보관함에서 과거 기간의 기록 계속 열람 가능
- 학부모 계정은 원아 기준 연결 → 반이동과 무관하게 기록 열람 유지

### 3.3 권한 즉시 해제
- 이동/삭제 시점부터 알림 수신·편집·삭제 권한 즉시 해제
- 교사: 해당 반에서 수행하던 알림 수신/편집/관리 권한 해제
- 원아: 새 반 기준으로 노출/대상 전환 (기존 반 리스트에서 제외)

### 3.4 담임 규칙
| 규칙 | 내용 |
|------|------|
| 담임 최소 1명 | 담임 0명 되는 삭제/이동 차단 → "담임 지정하기" 유도 |
| 담임 복수 허용 | 최대 2명까지 |
| 교사 최대 5명 | 한 반에 교사 5명 초과 시 추가 불가 + 안내 토스트 |

### 3.5 반 생성 초기 상태
- 교사 0명, 원아 0명, 운영 상태 N(비운영)

## 4. 화면 구성

### 4.1 반 목록

#### KPI 카드 (4개)
| 카드 | 색상 | 내용 |
|------|------|------|
| 전체 반 | 액센트 | 전체 반 수 (비운영 포함) |
| 운영중 반 | 초록 | status === 'active' 수 |
| 총 원아 수 | 보라 | childCount 합계 |
| 평균 충원율 | 앰버 | (총 원아 / 총 정원) × 100% |

#### 필터 / 액션바
- 연도 선택 드롭다운 (학년도 기준, 예: 2025년)
- 통합 검색: 반명, 원아명, 교직원명으로 검색
- 상태 필터 (전체 / 운영중 / 비운영)
- 검색 결과 건수 표시
- [+ 반 추가] 버튼 (super/org_admin만)

#### 반 카드 그리드 (3열)
각 카드 표시 항목:
- 상단 컬러바 (반마다 고유 색상)
- 반 이름 + 연령 (예: 송이반(5세))
- 운영 상태 배지 (Y/N)
- 교사 수 / 원아 수 요약
- 담당 교사명 (예: 강하나 교사 / 박하나 교사)
- 액션 버튼: [수정] / [관리] (권한 분기)

### 4.2 반 상세 드로어 (카드 클릭 → 관리)

#### 헤더
- 반명(연령) (예: 송이반(5세))
- 운영 상태 토글 (Y/N 전환 가능, super/org_admin만)

#### 교사 섹션
- 섹션 타이틀: `교사(n)` + [+ 교사 추가] 버튼
- 교사 카드 (가로 배열):
  - 프로필 + 이름
  - 역할 라벨 (담임/보조)
  - 액션 버튼 2개: [반이동] / [삭제]
- 교사 5명일 때 [+ 교사 추가] 비활성 → 클릭 시 토스트: "한 반에 교사는 최대 5명까지 추가할 수 있어요."

#### 원아 섹션
- 섹션 타이틀: `원아(n)` + [+ 원아 추가] 버튼
- 원아 검색 입력
- 원아 리스트 (세로):
  - 프로필/아바타 + 이름
  - 생년월일
  - 액션 버튼 2개: [반이동] / [삭제]

## 6. 교사 관리

### 6.1 교사 추가 모달

> 앱: 풀 바텀시트 → 웹: 모달

- **타이틀**: "{반명} 교사 추가"
- **검색**: 교사명으로 검색 (실시간 필터링)
- **교사 리스트** (복수 선택):
  - 프로필 + 교사명 + 역할(담임/보조) + 소속반 수
  - 이미 해당 반 소속 교사 → 선택 불가(비활성 처리)
  - 선택된 교사 → 체크 표시 + 하단 칩 영역에 표시
- **선택 칩 영역**: 선택된 교사 칩 (× 클릭으로 해제, 리스트와 동기화)
- **CTA**: 취소 / 추가(n)
  - 선택 0명: 추가 버튼 비활성
  - 추가 클릭 → **확인 모달**: "{반명}에 교사를 n명 추가하시겠습니까?" + 교사명 리스트
- **성공 처리**: 교사 카드 추가 + 카운트 갱신

### 6.2 교사 삭제

#### 6.2.1 일반 삭제 (보조 교사 또는 담임이 2명 이상 남는 경우)
- **트리거**: 반 상세 > 교사 카드 [삭제] 클릭
- **모달**: "{교사명} 교사({역할})를 {반명}에서 삭제할까요?"
- **안내**: "삭제 후에도 {반명}에 남긴 기록은 그대로 유지되나, 알림 수신 및 편집·삭제 권한은 즉시 해제됩니다."
- **CTA**: 취소 / 삭제
- **성공**: 교사 카드 제거 + 카운트 갱신 + 토스트: "{교사명} 교사를 {반명}에서 삭제했어요."

#### 6.2.2 담임 삭제 제한 (담임 0명 방지)
- **조건**: 삭제 대상이 담임이고, 삭제 후 담임 0명이 되는 경우
- **모달**: "담임 교사를 먼저 지정해 주세요"
- **안내**: "{반명}에는 담임이 최소 1명 필요해요. 다른 교사를 담임으로 지정한 후 {교사명} 교사를 반에서 제외할 수 있어요."
- **CTA**: 취소 / 담임 지정하기 → 역할 변경 모달로 이동

### 6.3 역할 변경 모달

> 앱: 바텀시트 → 웹: 모달

- **타이틀**: "{반명} 역할 선택"
- **검색**: 교사명으로 검색 (실시간 필터링)
- **교사 리스트**:
  - 프로필 + 교사명 + 소속반 수
  - 각 행에 역할 선택: 담임 / 보조
  - 현재 역할을 초기값으로 표시
- **역할 선택 규칙**:
  - 저장 시점에 담임 0명이면 저장 불가
  - 담임 복수 허용 (최대 2명)
  - 변경 시 즉시 화면 내 배지 갱신 (저장 전 임시 상태)
- **CTA**: 취소 / 저장
  - 변경사항 없으면 저장 비활성
- **트리거**:
  - A: 담임 교사 삭제 시도 → "담임 지정하기" CTA
  - B: 교사 카드의 [역할 변경] 메뉴
  - C: 담임 교사 반이동 시도 → "담임 지정하기" CTA

### 6.4 교사 반이동 모달

- **트리거**: 반 상세 > 교사 카드 [반이동] 클릭
- **화면**: 대상 교사 프로필 + 이름 + 역할(담임/보조)
- **반 목록**:
  - 현재 소속 반 → "소속반" 배지 (선택 불가)
  - 이동 가능한 반 → 반명(연령) + 교사 수 + [이동] 버튼
- **이동 확인 모달**: "{교사명} 교사를 {이전반} → {이동반}으로 이동할까요?"
  - 안내: "{이전반}의 기록은 유지되며, 편집·삭제 권한은 모두 해제됩니다."
  - CTA: 취소 / 이동
- **담임 이동 시**: 담임 0명 되면 이동 차단 → "담임 지정하기" 유도 (§6.2.2와 동일)
- **성공 토스트**: "{교사명} 교사가 {이동반}으로 이동되었습니다. (이전 반 권한 해제)"

## 7. 원아 관리

### 7.1 원아 추가 모달

> 앱: 풀 바텀시트 → 웹: 모달

- **타이틀**: "{반명} 원아 추가"
- **검색**: 원아명으로 검색 (실시간 필터링)
- **정렬/필터**: 가나다순 정렬 + 연령 필터 드롭다운
- **원아 리스트** (복수 선택):
  - 프로필 + 원아명 + 연령(또는 생년월일) + 소속반
  - 이미 해당 반 등록 원아 → 선택 불가(비활성)
  - 선택된 원아 → 체크 표시 + 하단 칩 영역
- **선택 칩 영역**: 선택된 원아 칩 (× 클릭으로 해제)
- **CTA**: 취소 / 추가(n) (선택 0명이면 비활성)
- **성공 처리**: 원아 리스트에 추가 + 카운트 갱신 + 추가된 원아 카드 일시적 강조(하이라이트)

### 7.2 원아 삭제 (반에서 제외)

- **트리거**: 반 상세 > 원아 카드 [삭제] 클릭
- **모달**: "{원아명} 원아를 {반명}에서 제외할까요?"
- **안내**: "기록은 유지되며, 해당 반에서만 제외됩니다."
- **CTA**: 취소 / 제외
- **성공**: 원아 카드 제거 + 카운트 갱신 + 토스트

### 7.3 원아 반이동 모달

- **트리거**: 반 상세 > 원아 카드 [반이동] 클릭
- **화면**: 대상 원아 프로필 + 이름 + 생년월일/연령
- **반 목록**:
  - 현재 소속 반 → "소속반" 배지 (선택 불가)
  - 이동 가능한 반 → 반명(연령) + [이동] 버튼
- **이동 확인 모달**: "{원아명} 원아를 {현재반} → {대상반}으로 이동할까요?"
  - 안내: "이전 반의 기록은 유지됩니다."
  - CTA: 취소 / 이동
- **성공 토스트**: "{원아명} 원아가 {대상반}으로 이동되었습니다."

## 11. 통합 검색 정책

교사/원아명으로 검색 시 소속 반 리스트를 출력한다.

```
예) "강하나" 검색
→ 강하나 교사 — 송이반, 예리반
→ 강하나 원아 — 나무반
→ 결과: 송이반, 예리반, 나무반 리스트 노출
```

- 동명이인 시 역할(교사/원아) 구분 표시

## 12. 미결 사항

| 항목 | 내용 |
|------|------|
| 반 색상 자동 배정 | 신규 반 추가 시 고유 색상 풀에서 순환 배정 |
| 반 복사 기능 | 이전 학년도 반 구조 복사 기능 필요 여부 |
| 동명이인 검색 노출 | 교사/원아 동명이인 시 노출 방식 확정 필요 |
| 원아 겸임/중복 정책 | 원아 추가 시 기존 다른 반 소속이 있는 경우 처리 기준 (겸임/이동/중복) |
| 정원 관리 | 앱 기획에서 정원 필드가 명시되지 않음 — 웹에서 유지할지 결정 필요 |
