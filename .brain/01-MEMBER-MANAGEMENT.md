## 1. 통합회원 조회 (`member-list-admin.html`)

### 1.1 기능 개요
- 시스템에 등록된 **모든 회원**을 통합적으로 조회하는 화면
- 교사, 학부모, 기관장 전체를 하나의 리스트로 관리

### 1.2 KPI 카드 (상단)
| 카드 | 설명 |
|------|------|
| 전체 회원 수 | 시스템 전체 등록 회원 |
| 교사 회원 수 | 교사+기관장 합산 |
| 학부모 회원 수 | 학부모 계정 수 |
| 활성률 카드 (추정) | 재직중/재원중 비율 |

### 1.3 테이블 필드 (회원 데이터 모델)
```javascript
{
  status: String,       // 's-active'|'s-enroll'|'s-grad'|'s-leave'|'s-out'|'s-stop'
  statusLabel: String,  // '재직중'|'재원중'|'졸업'|'퇴소/퇴사'|'서비스탈퇴'|'정지'
  type: String,         // 'teacher'|'parent'|'director'
  typeLabel: String,    // '교사'|'학부모'|'기관장'
  guardian: String,     // '-'|'주계정'|'가족계정'
  name: String,         // 회원 이름
  phone: String,        // 연락처 (010-XXXX-XXXX)
  affiliMain: String,   // 소속 기관명
  affiliSub: String,    // 소속 상세 (반/원아 정보)
  conn: Number|'-',     // 다중기관 연결 수
  role: String,         // 역할 (담임, 보조, 엄마, 아빠 등)
  pos: String,          // 직위 (원장, 부원장, 교사, 보조교사 등)
  route: String,        // 가입경로: '기존통합회원'|'초대장'|'직접가입'
  login: String,        // 로그인 방식: '통합ID'|'SNS-카카오'|'SNS-네이버'|'SNS-구글'|'SNS-애플'
  join: String,         // 가입일 (YYYY-MM-DD)
  last: String,         // 최근 접속일 (YYYY-MM-DD)
  avatar: String,       // 아바타 그라디언트 색상 (예: '#4361EE,#7C3AED')
  avatarChar: String    // 아바타 표시 글자 (성씨)
}
```

### 1.4 상태 컬러 시스템
| 상태 | 배경색 | 텍스트색 | 도트색 |
|------|--------|----------|--------|
| 재직중 | #D1FAE5 | #065F46 | #10B981 |
| 재원중 | #DBEAFE | #1E40AF | #3B82F6 |
| 졸업 | #EDE9FE | #5B21B6 | #8B5CF6 |
| 퇴소/퇴사 | #FEF3C7 | #92400E | #F59E0B |
| 서비스탈퇴 | #FEE2E2 | #991B1B | #EF4444 |
| 정지 | #F3F4F6 | #374151 | #9CA3AF |

### 1.5 필터/검색
- **회원유형 필터**: 전체, 교사, 학부모, 기관장
- **상태 필터**: 전체, 재직중, 재원중, 졸업, 퇴소/퇴사, 서비스탈퇴
- **가입경로 필터**: 전체, 기존통합회원, 초대장, 직접가입
- **보호자 필터**: 전체, 주계정, 가족계정 토글
- **텍스트 검색**: 이름, 기관명, 연락처 검색
- **상태 카드 클릭 필터**: 상단 KPI 카드 클릭 시 해당 상태로 필터

### 1.6 가입경로/로그인 태그 스타일
| 태그 | 스타일 클래스 |
|------|--------------|
| 기존통합회원 | `tag-integrated` (보라) |
| 초대장 | `tag-invite` (파랑) |
| 직접가입 | `tag-direct` (회색) |
| 통합ID | `tag-unified` (진한 보라) |
| SNS-카카오 | `tag-kakao` (노랑) |
| SNS-네이버 | `tag-naver` (초록) |
| SNS-구글 | `tag-google` (멀티) |
| SNS-애플 | `tag-apple` (검정) |

## 2. 가입/승인 관리 (`member-approval.html`)

### 2.1 기능 개요
- 가입 신청 건을 **승인/거절** 처리하는 화면
- 승인대기, 승인완료, 승인거절 3가지 상태 관리

### 2.2 KPI 카드
| 카드 | 색상 | 설명 |
|------|------|------|
| 승인대기 | 파랑 (#3B82F6) | 아직 처리되지 않은 건수 |
| 승인완료 | 초록 (#10B981) | 승인 처리된 건수 |
| 승인거절 | 빨강 (#EF4444) | 거절 처리된 건수 |

### 2.3 가입승인 데이터 모델
```javascript
{
  status: String,       // 'pending'|'approved'|'rejected'
  type: String,         // 'teacher'|'parent'|'director'
  name: String,         // 신청자명
  org: String,          // 기관명
  orgType: String,      // '유치원'|'어린이집'
  cls: String,          // 반 정보 (예: '꽃잎반(2세)')
  role: String,         // 역할 (담임교사, 보조교사, 엄마, 아빠 등)
  route: String,        // 가입경로: '기존통합회원'|'초대장'|'직접가입'
  login: String,        // 로그인방식: '통합ID'|'SNS-카카오'|'SNS-네이버'|'SNS-구글'
  inv: Boolean,         // 초대장 수신 여부 (O/X)
  reqDate: String,      // 신청일시 (YYYY-MM-DD HH:mm)
  approvedDate: String, // 처리일시
  canCancel: Boolean,   // 승인취소 가능 여부
  phone: String,        // 연락처
  email: String,        // 이메일
  gender: String,       // 성별
  birth: String,        // 생년월일
  guardian: String,     // 보호자 유형 ('—'|'주계정')
  conn: Number          // 다중기관 연결 수
}
```

### 2.4 승인 정책
| 규칙 | 설명 |
|------|------|
| 승인대기 → 승인 | 즉시 승인 처리 (확인 모달) |
| 승인대기 → 거절 | 거절 사유 선택 필요 (확인 모달) |
| 승인완료 → 취소 | `canCancel=true`인 경우만 가능 |
| 승인거절 상태 | 추가 액션 불가 |
| 일괄 처리 | 체크박스 선택 → 일괄 승인/거절 버튼 |

### 2.5 필터
- **상태 필터**: 전체, 승인대기, 승인완료, 승인거절
- **유형 필터**: 전체, 교사, 학부모
- **가입경로 필터**: 전체, 기존통합회원, 초대장, 직접가입
- **텍스트 검색**: 이름, 기관명

### 2.6 상세 패널
- 우측 슬라이드 패널로 상세 정보 표시
- 기본정보, 기관/반 정보, 계정정보 탭

## 3. 회원 상태 관리 (`member-status-admin.html`)

### 3.1 기능 개요
- 회원의 **서비스 상태를 변경**하는 관리 화면
- 정상, 정지/휴면, 서비스탈퇴 상태 전환

### 3.2 상태 체계
| 상태 | 배경색 | 텍스트색 | 설명 |
|------|--------|----------|------|
| 정상 | #D1FAE5 | #065F46 | 활성 상태 |
| 정지/휴면 | #FEF3C7 | #92400E | 서비스 이용 일시 정지 |
| 서비스탈퇴 | #FEE2E2 | #991B1B | 서비스 완전 탈퇴 |
| 졸업 | #EDE9FE | #5B21B6 | 원아 졸업 후 상태 |

### 3.3 상태 변경 모달
- 변경 대상 상태 선택 (라디오 옵션)
- 사유 입력 (텍스트영역, 글자수 카운트)
- 서비스탈퇴 시 경고 메시지 표시

### 3.4 탭 구성
- **전체** / **정상** / **정지·휴면** / **서비스탈퇴** 탭

### 3.5 일괄 처리
- 체크박스로 복수 선택 후 일괄 상태 변경
- 드롭다운 메뉴: 상세보기, 상태변경 등

## 4. 회원 관련 비즈니스 규칙 요약

### 4.1 회원 유형
| 유형 | 코드 | 설명 |
|------|------|------|
| 교사 | `teacher` | 기관 소속 교사 |
| 학부모 | `parent` | 원아의 보호자 |
| 기관장 | `director` | 원장/부원장 |

### 4.2 보호자 계정 구분
| 유형 | 설명 |
|------|------|
| 주계정 | 최초 가입한 보호자 (엄마 또는 아빠) |
| 가족계정 | 주계정에 연결된 가족 (배우자 등) |
| — | 해당 없음 (교사/기관장) |

### 4.3 가입 경로
| 경로 | 설명 |
|------|------|
| 기존통합회원 | 키드키즈 기존 통합 계정에서 연동 |
| 초대장 | 기관에서 발송한 초대 링크를 통한 가입 |
| 직접가입 | 앱/웹에서 직접 회원가입 |

### 4.4 로그인 방식
| 방식 | 설명 |
|------|------|
| 통합ID | 키드키즈 통합 ID/PW |
| SNS-카카오 | 카카오톡 소셜 로그인 |
| SNS-네이버 | 네이버 소셜 로그인 |
| SNS-구글 | 구글 소셜 로그인 |
| SNS-애플 | 애플 소셜 로그인 |

### 4.5 다중기관 연결 (conn)
- 한 회원이 여러 기관에 동시 소속될 수 있음
- 예: 교사가 2개 기관에 근무, 학부모가 자녀 2명 다른 기관
