## 개요

기관 단위 급식 식단표를 등록·조회하는 화면.  
**날짜 = 1개의 게시글** 단위로, 하루에 최대 4개 식사구분(오전간식·점심·오후간식·석식)을 입력한다.  
반별 구분 없이 **공통 식단** 단일 구조로 운영된다.  
학부모 앱에 공유된다.

## 레이아웃 구조

```
[페이지]
├─ 상단 헤더
│   ├─ 페이지 타이틀 "식단표"
│   └─ [+ 식단 등록] 버튼 (우측)
│
└─ 콘텐츠 영역
    ├─ 풀위스 월간 달력 (항상 표시)
    │   ├─ 달력 헤더: < 2026년 04월 >
    │   └─ 날짜 셀 (등록된 식사구분 태그 + 사진 아이콘 표시)
    │
    └─ 날짜 콘텐츠 패널 (달력 클릭 시 달력 아래에 표시)
        ├─ 패널 헤더: 선택 날짜 + [수정] [삭제] 버튼
        └─ 등록된 식사구분 조회 영역 (오전간식→점심→오후간식→석식 순서 고정)
```

---

## 핵심 기능

### 1. 풀위스 월간 달력

- 페이지 전체 너비를 차지하는 달력 단일 뷰 (피드 뷰 없음)
- 상단 `< 2026년 04월 >` 네비게이션으로 월 이동
- **페이지 진입 시 오늘 날짜가 자동 선택**되어 콘텐츠 패널 즉시 표시

#### 날짜 셀 표시

| 상태 | 표시 |
|------|------|
| 등록된 식사구분 있음 | 식사구분 태그 (해당하는 것만) + 사진 아이콘 (사진 등록 시) |
| 등록된 식사구분 없음 | 날짜 숫자만 표시, 호버 시 [+ 등록하기] 버튼 표시 |
| 선택된 날짜 | 선택 강조 스타일 |

#### 식사구분 태그 색상

| 식사구분 | 배경 | 텍스트 |
|---------|------|--------|
| 오전간식 | `#FFF7ED` | `#C2410C` (주황) |
| 점심 | `#EFF6FF` | `#1D4ED8` (밝은파랑) |
| 오후간식 | `#F5F3FF` | `#6D28D9` (보라) |
| 석식 | `#EEF2FF` | `#3730A3` (남색) |

> 사진 아이콘: 📷 또는 카메라 아이콘 (식사구분 태그 우측에 소형)

---

### 2. 날짜 콘텐츠 패널

달력 아래 고정 영역. 날짜 클릭 시 해당 날짜 내용으로 교체.

#### 등록된 날짜 클릭 시

```
[2026년 4월 15일 (수)]  [수정]  [삭제]
────────────────────────────────────────
오전간식
  메뉴: 찐빵, 우유
  사진: [썸네일]

점심
  메뉴: 잡곡밥, 미역국, 돈까스
  사진: [썸네일]

오후간식
  메뉴: (미등록 — 표시 안 함)

석식
  메뉴: (미등록 — 표시 안 함)
```

> 미등록 식사구분은 패널에서 표시하지 않음.

#### 미등록 날짜 클릭 시

```
[2026년 4월 20일 (월)]
────────────────────────────────────────
등록된 식단이 없습니다.
[+ 식단 등록하기] 버튼
```

---

### 3. 식단 등록/수정 모달

진입 경로:
- 상단 `[+ 식단 등록]` 버튼 클릭
- 달력 빈 날짜 셀 호버 → `[+ 등록하기]` 버튼 클릭
- 콘텐츠 패널 `[수정]` 버튼 클릭 (해당 날짜 pre-fill)

#### 모달 구조

```
[식단 등록] / [식단 수정]                         ×
────────────────────────────────────────────────
날짜      [날짜 picker]

────── 오전간식 ──────────────────────────────
메뉴 입력  [textarea]
사진       [이미지 업로드 영역 (선택사항)]
           등록된 이미지 썸네일 + [삭제] 버튼

────── 점심 ──────────────────────────────────
메뉴 입력  [textarea]
사진       [이미지 업로드 영역 (선택사항)]

────── 오후간식 ──────────────────────────────
메뉴 입력  [textarea]
사진       [이미지 업로드 영역 (선택사항)]

────── 석식 ──────────────────────────────────
메뉴 입력  [textarea]
사진       [이미지 업로드 영역 (선택사항)]

────────────────────────────────────────────────
                          [취소]  [저장]
```

#### 모달 유효성

- **최소 1개** 식사구분에 메뉴가 입력되어야 저장 가능
- 사진은 식사구분별 1장, 선택사항
- 수정 시 기존 이미지를 삭제만 하고 저장 가능 (이미지 없는 상태 허용)
- 같은 날짜에 중복 등록 불가 (수정 모달로 진입)
- 미래 날짜 등록 가능

---

### 4. 식단 삭제

- 콘텐츠 패널 `[삭제]` 버튼 클릭
- 확인 모달 표시: "해당 날짜의 식단을 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다."
- **날짜 전체 삭제** (식사구분 단위 삭제 없음)
- 삭제 후 콘텐츠 패널은 미등록 상태로 전환

## 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 해당 월에 등록된 식단 없음 | 달력 날짜 셀에 태그 없음. 오늘 날짜 패널에 "등록된 식단이 없습니다." |
| 이미 등록된 날짜에 등록 시도 | 달력 셀 호버 시 [등록하기] 대신 [수정하기] 버튼 표시 (또는 자동으로 수정 모달) |
| 4개 식사구분 모두 비워두고 저장 | "최소 1개 식사구분을 입력해주세요." 토스트/인라인 오류 |
| 달력 월 이동 후 이전 선택 날짜 | 선택 해제, 콘텐츠 패널 초기화 (오늘 날짜 자동 선택 없음) |
