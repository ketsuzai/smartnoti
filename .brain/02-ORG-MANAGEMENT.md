# 🧠 프로젝트 브레인: 기관 관리 기능정의

> **최종 업데이트**: 2026-03-18  
> **관련 파일**: `org-admin.html`

---

## 1. 기관관리 개요

- 3가지 탭(기관 조회 / 기관 승인 관리 / 기관 상태 관리)으로 구성
- URL 해시(`#approval`, `#status`)로 탭 직접 접근 가능

---

## 2. 기관 데이터 모델

```javascript
{
  id: Number,              // 기관 고유 ID
  name: String,            // 기관명
  type: String,            // '어린이집'|'유치원'
  director: String,        // 대표자(원장) 이름
  phone: String,           // 기관 연락처
  region: String,          // 지역: '서울'|'경기'|'인천'|'부산'|'대구'|'기타'
  cert: String,            // 인증 상태: '인증완료'|'인증대기'|'인증실패'
  approval: String,        // 승인 상태: '승인완료'|'승인대기'|'반려'
  status: String,          // 이용 상태: '이용중'|'이용종료'|'승인대기'|'반려'
  membership: String,      // 멤버십: '멀티회원'|'정회원'|'일반회원'|'미가입'
  memberPlan: String,      // 가입 플랜명
  memberSince: String,     // 멤버십 시작일 (YYYY.MM.DD)
  memberExpire: String,    // 멤버십 만료일 (YYYY.MM.DD 또는 '-')
  cls: Number,             // 반 수
  teacher: Number,         // 교사 수
  kid: Number,             // 원아 수
  joinDate: String,        // 기관 가입일
  lastDate: String,        // 최근 접속일
  biz: String,             // 사업자등록번호

  // 멀티회원 전용 (옵션)
  multiMembers: Array<{    // 멀티회원 소속 교사 목록
    id: String,
    name: String,
    email: String,
    role: String,           // '담임교사'|'보조교사'
    joinedAt: String,
    active: Boolean,
    avatarBg: String
  }>
}
```

## 3. 탭 1: 기관 조회

### 3.1 KPI 카드
| 카드 | 설명 |
|------|------|
| 전체 기관 | 전체 등록 기관 수 |
| 이용중 | 현재 서비스 이용중인 기관 |
| 멀티회원 | 멀티회원 등급 기관 |
| 정회원 | 정회원 등급 기관 |
| 일반회원 | 일반회원 등급 기관 |

### 3.2 테이블 컬럼
`체크박스 | 기관명 | 유형 | 대표자 | 연락처 | 지역 | 인증상태 | 승인상태 | 반수 | 교사수 | 원아수 | 멤버십 | 이용상태 | 가입일 | 최근접속 | 상세`

### 3.3 필터
- **유형**: 전체, 어린이집, 유치원
- **지역**: 전체, 서울, 경기, 인천, 부산, 대구, 기타
- **멤버십**: 전체, 멀티회원, 정회원, 일반회원, 미가입
- **이용상태**: 전체, 이용중, 이용종료
- **텍스트 검색**: 기관명, 대표자

---

## 4. 탭 2: 기관 승인 관리

### 4.1 대상 데이터
- `approval === '승인대기'` 또는 `approval === '반려'` 인 기관만 표시

### 4.2 테이블 컬럼
`체크박스 | 기관명 | 유형 | 대표자 | 연락처 | 지역 | 사업자번호 | 인증상태 | 승인상태 | 가입일 | 액션`

### 4.3 승인 액션
| 액션 | 조건 | 설명 |
|------|------|------|
| 승인 | 승인대기 상태 | 기관을 승인 처리 |
| 반려 | 승인대기 상태 | 기관을 반려 처리 |
| 상세보기 | 모든 상태 | 기관 상세 정보 패널 |

---

## 5. 탭 3: 기관 상태 관리

### 5.1 대상 데이터
- `membership !== '미가입'` 이고 `approval === '승인완료'`인 기관

### 5.2 테이블 컬럼
`기관명 | 유형 | 대표자 | 반수 | 교사수 | 원아수 | 멤버십 | 이용상태 | 가입일 | 최근접속 | 관리`

### 5.3 관리 액션
- 기관 상세보기
- 이용 상태 변경 (이용중 ↔ 이용종료)

---

## 6. 기관 상세 패널 (Side Panel)

### 6.1 기본 정보 섹션
- 기관명, 유형, 지역, 연락처, 대표자, 사업자번호

### 6.2 멤버십 정보 섹션
- 멤버십 등급, 가입 플랜, 시작일, 만료일

### 6.3 운영 정보 섹션
- 반 수, 교사 수, 원아 수, 가입일, 최근 접속

### 6.4 멀티회원 전용: 소속 교사 목록
- 활성/비활성 교사 리스트
- 교사별 이름, 이메일, 역할, 가입일
