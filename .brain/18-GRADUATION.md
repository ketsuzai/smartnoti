# 18-GRADUATION.md — 진급/졸업

> 상태: ✅ 완료 (2026-04-19 — 필터 정적화, 헤더 컴포넌트 적용)
> 원아의 연말 진급(반 이동) 및 졸업 처리를 일괄로 수행하는 화면.
> 파일: `src/pages/oper/operation-graduation.html`

---

## 개요

새 학년도 시작 시(2월~3월) 원아별 진급/졸업 상태를 자동 제안받고 확정 처리하는 배치성 화면.
- 기관관리자(`org_admin`) 필수, 교사(`teacher`)는 담당반만 접근 가능
- 진급 처리 완료 시 원아 status를 `graduated`로 변경 (localStorage 동기화)
- 보류(확인필요) 상태로 변경 시 사유 입력 필수

## 처리 탭 기능

### 데이터 모델 v2 (2026-04-20)
- **처리 리스트 소속 = `!graduation_status || graduation_status === 'on_hold'`**
- 보류 원아는 처리 탭에서 편집 대상으로 유지 (완료 리스트에는 표시되지 않음)
- KPI "미확정" 카운트도 보류 포함

### 테이블 컬럼
| 항목 | 설명 |
|------|------|
| 체크박스 | 헤더 전체선택(+indeterminate) / 행 단위 선택 |
| 원아(이름/생년월일) | 아바타 + 이름 (profile_color) / 생년월일 (회색) |
| 소속반(25년) | 현재 반명 |
| 처리상태 | 미확정 배지 / 보류 배지 + 보류사유 표시 |
| 자동제안결과 | 드롭다운: 진급 \| 졸업 \| 보류 — 기본값 `statusChanges → graduation_status → graduation_suggestion` 순 |

### 필터 & 액션
- **상태 select**: 전체 / 진급대상 / 졸업대상 / 보류 *(칩 방식 → select 드롭다운으로 변경, 2026-04-19)*
- **나이 select** · **반 select** · **이름 input** · 초기화 버튼 (한 줄 filter-row)
- **일괄 버튼**: 전체선택 · 선택진급 · 선택졸업 · 보류처리 · 자동제안 전체적용
  - 선택진급: 체크된 원아 모두 진급으로 일괄 변경
  - 선택졸업: 체크된 원아 모두 졸업으로 일괄 변경
  - 보류처리: 체크된 원아 모두 보류로 변경 → 사유 입력 모달 (순차 입력 또는 일괄)

### 하단 선택 요약 바 (복수 선택 시 노출)
```
선택 n명 | 진급 a / 졸업 b / 보류 c  [변경/취소]
```

## 완료 탭 기능

### 테이블 컬럼
| 항목 | 설명 |
|------|------|
| 체크박스 | 헤더 전체선택(+indeterminate) / 행 단위 선택 |
| 원아(이름/연령) | 이름 + 만 나이 |
| 소속반(26년) | 새 학년도 배정된 반명 (또는 "배정 대기") |
| 진급상태 배지 | 진급/졸업 배지 + 행별 `(변경)` 버튼 |
| 처리자 / 처리일시 | 확정 이력 |

### 필터
- **상태 select**: 전체 / 진급 / 졸업 *(칩 방식 → select 드롭다운으로 변경, 2026-04-19)*
- **보류는 여기서 제외** — 보류 원아는 처리 탭에서만 노출됨 (데이터 모델 변경 v2)

### 액션바 (처리 탭과 동일 구조)
- [좌] 총 n건 · 선택 n명
- [우] 전체선택 · 선택진급 · 선택졸업 · 보류처리
- **선택진급/선택졸업**: 확정된 상태를 다른 상태로 재처리
- **보류처리**: 완료 항목을 보류로 돌려 처리 탭으로 이동 → 사유 입력 필수
  - 확정 후 처리 탭으로 자동 전환 + 보류 필터 적용 + 1.5초 노란 하이라이트
- **행별 `(변경)`**: 단건 재편집 진입점 (기존 유지)

## 진급상태 변경 모달

### 구성
- 제목: "진급상태 변경/취소"
- 원아별 행 (각 원아 1줄)
  - 원아명 + 드롭다운 (진급 \| 졸업 \| 보류)
  - 보류 선택 시 사유 필드 인라인 표시:
    ```
    ┌─────────────────────────┐
    │ 보류 사유:  [셀렉트박스]  │  (서류미비/연령확인필요/반미개설/기타)
    │          [메모 입력]     │  (기타 선택 시만 필수)
    └─────────────────────────┘
    ```
- 키보드 덮침 주의: 사유 메모 포커스 시 모달이 뷰포트 기준 상단으로 자동 스크롤
- 버튼: [이전] [변경확정(n명)]

### 이탈 경고
- [이전] 클릭 시 드롭다운 변경 감지 → 모달 팝업:
  ```
  "[이전]으로 돌아갈경우,
   변경사항이 반영되지 않습니다.
   계속 진행하시겠습니까?"
  [취소] [계속진행]
  ```

## 확정 처리 플로우

1. **처리 탭에서 체크 → [변경/취소] → 상태 변경 모달 열기**
   - 원아별로 드롭다운 변경
   - 보류 선택 시 사유 입력
   - [변경확정(n)] 클릭

2. **확정 처리 확인 모달**
   ```
   "진급/졸업처리 확정을 진행하시겠습니까?"
   [취소] [확정(n)]
   ```

3. **진행 중 오버레이** (1~2초)
   ```
   진급상태 변경 적용 중
   (2/3)  ← 진행 카운트
   [스피너]
   ```

4. **완료 컨펌창** (2가지 분기)
   - **A. 새 학년도 반 없을 때**:
     ```
     ✅ 진급/졸업 처리가 완료 되었어요.
     새 학년도에 아직 반이 없어요.
     배정을 하려면 먼저 반을 만들어 주세요.
     [나중에하기] [반 개설하기]  → operation-class.html
     ```
   - **B. 새 학년도 반 있을 때**:
     ```
     ✅ 진급/졸업 처리가 완료 되었어요.
     새 학년도에 개설된 반으로 이제 원아를 배정할 수 있어요.
     [나중에하기] [원아 배정하기]  → operation-child.html
     ```

## 라우팅 로직 (처리 후 다음 단계)

### Case 1: 보류가 1건이라도 포함된 경우
→ 완료 컨펌창 [확인] 클릭 후 **처리 탭으로 자동 복귀**
→ 필터를 **보류로 자동 활성**
→ 보류로 바뀐 원아 1~2초 하이라이트

### Case 2: 진급/졸업만 변경된 경우
→ 완료 컨펌창 [확인] 클릭 후 **완료 탭 유지**
→ 테이블만 새로고침

## 데이터 구조 & localStorage 동기화

### 모킹 데이터 필드 (MOCK_CHILDREN 확장)
```js
{
  id: 'CHD001',
  name: '강가영',
  birth: '240628',
  age: '2세',
  class_id: 'CLS001',
  class_name: '꽃잎반',
  status: 'enrolled',  // 재원중/졸업/퇴소/휴원
  
  // ═══ 진급/졸업 필드 ═══
  graduation_status: 'promoted' | 'graduated' | 'on_hold' | 'promoted_auto' | null,
  graduation_suggestion: 'promoted' | 'graduated',  // 자동 제안
  hold_reason: '서류미비' | '연령확인필요' | '반미개설' | '기타',
  hold_memo: '',
  next_class: { id: 'CLS002', name: '나무반', age: '3세' }  // 새 학년도 배정 반
}
```

### localStorage 연동
```js
// 확정 처리 시
const GRAD_KEY = 'graduation_confirmed';
const confirmed = { ids: ['CHD001', 'CHD002'], timestamp: Date.now() };
localStorage.setItem(GRAD_KEY, JSON.stringify(confirmed));

// operation-member.html에서 init() 시
const grad = JSON.parse(localStorage.getItem(GRAD_KEY) || '{}');
if (grad.ids?.length > 0) {
  MOCK_CHILDREN.forEach(c => {
    if (grad.ids.includes(c.id)) {
      c.status = 'graduated';  // status 업데이트
    }
  });
}
```

## 운영 로직

### 노출 트리거 (언제 화면을 보여줄까?)
다음 중 **하나라도 true**면 대시보드 배너 + 사이드바 알림 표시:
- 새 학년도 반이 하나도 생성되지 않음

## 주의사항

- 진급/졸업 실행은 되돌리기 어려운 작업 → 확인 모달 필수
- 원아 status를 `graduated`로 변경하면 operation-member.html에서도 자동 반영 (localStorage)
- 보류 사유 입력 필수 → 비워두면 toast 에러
- 교사는 담당반만, 진급 후 소속반 없는 원아는 모든 교사에게 노출
